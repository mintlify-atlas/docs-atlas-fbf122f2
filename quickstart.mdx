---
title: "Quick start"
description: "Get the Airplane Tracker MCP Server running with Claude Desktop in under 5 minutes."
---

This guide uses the Python installation path. If you prefer Docker, see the [Docker installation guide](/installation/docker).

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Bellaposa/airplanes-live-mcp.git
    cd airplanes-live-mcp
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    A virtual environment is required. Without it, Claude Desktop cannot locate the installed dependencies.

    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        python -m venv .venv
        source .venv/bin/activate
        ```
      </Tab>
      <Tab title="Windows">
        ```bash
        python -m venv .venv
        .venv\Scripts\activate
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs `mcp[cli]`, `fastmcp`, and `httpx`.
  </Step>

  <Step title="Test the server">
    Run the server to confirm it starts without errors:

    ```bash
    python airplane_server.py
    ```

    The server starts and waits for MCP input on stdin. Press `Ctrl+C` to stop it.
  </Step>

  <Step title="Configure Claude Desktop">
    Open your Claude Desktop configuration file and add the `airplanes-live` server entry.

    <Tabs>
      <Tab title="macOS">
        File: `~/Library/Application Support/Claude/claude_desktop_config.json`

        ```json
        {
          "mcpServers": {
            "airplanes-live": {
              "command": "/path/to/airplanes-live-mcp/.venv/bin/python",
              "args": ["/path/to/airplanes-live-mcp/airplane_server.py"],
              "env": {
                "PYTHONPATH": "/path/to/airplanes-live-mcp"
              }
            }
          }
        }
        ```
      </Tab>
      <Tab title="Linux">
        File: `~/.config/claude-desktop/config.json`

        ```json
        {
          "mcpServers": {
            "airplanes-live": {
              "command": "/path/to/airplanes-live-mcp/.venv/bin/python",
              "args": ["/path/to/airplanes-live-mcp/airplane_server.py"],
              "env": {
                "PYTHONPATH": "/path/to/airplanes-live-mcp"
              }
            }
          }
        }
        ```
      </Tab>
      <Tab title="Windows">
        File: `%APPDATA%\Claude\claude_desktop_config.json`

        ```json
        {
          "mcpServers": {
            "airplanes-live": {
              "command": "C:\\Users\\YourUsername\\airplanes-live-mcp\\.venv\\Scripts\\python.exe",
              "args": ["C:\\Users\\YourUsername\\airplanes-live-mcp\\airplane_server.py"],
              "env": {
                "PYTHONPATH": "C:\\Users\\YourUsername\\airplanes-live-mcp"
              }
            }
          }
        }
        ```

        Replace `YourUsername` with your actual Windows username. Use double backslashes in all paths.
      </Tab>
    </Tabs>

    <Warning>
      Use the full absolute path to the `.venv` Python binary, not the system Python. If you point Claude Desktop at the system Python, it will fail to find the installed packages.
    </Warning>
  </Step>

  <Step title="Restart Claude Desktop and start tracking">
    Quit Claude Desktop completely and reopen it. The MCP server will start automatically when Claude connects.

    Try these queries to confirm everything is working:

    - "Show me aircraft near New York"
    - "Find flight UAL123"
    - "What military aircraft are currently flying?"
    - "Show aircraft with registration N12345"
    - "Find aircraft squawking 7700"
  </Step>
</Steps>

<Tip>
  If the tools don't appear in Claude after restarting, check that the paths in your config file are correct and that the virtual environment was created at `.venv` inside the cloned repository directory.
</Tip>
